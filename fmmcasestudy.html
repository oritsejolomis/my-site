AI-Assisted Location Discovery

Designing an intent-driven recommendation workflow for film production

Client

Filmmakers Mart

Timeline

Q1 – Q3 2025

Role

Snr. Product Designer

Output

Design System
Web Application

01 The Problem

Film production runs on creative intent. A director knows they want a location that feels isolated, industrial, post-Soviet — not a location tagged “warehouse.” The existing discovery tool only supported rigid filter-based search: category tags, location type, availability. Users had to translate creative ideas into database categories before they could even begin exploring.
This created a structural mismatch:

— The system was optimised for filtering, not creative decision-making

— Results were technically accurate but aesthetically irrelevant

— Discovery felt transactional — users got matches, not insight

— There was no way to express intent like mood, tone, or scene context

The core issue was not the filters themselves. It was that filtering was the only mode of discovery available.

02 Key Decision

Rather than improving the filter interface, I shifted the product direction toward AI-assisted contextual recommendations.
goal: let the system interpret production intent, not just match keywords.

Instead of matching locations by tags alone, the recommendation engine would evaluate:

— Aesthetic and visual similarity
— Emotional tone and scene context — Environmental fit (texture, density, atmosphere).
— Production practicality: lighting, accessibility, scale

This reframed the experience from:
“Find locations by category” → “Discover locations that fit the scene.”

03 What I Deliberately Avoided

The obvious move was a conversational chat interface. I rejected it.
A chat interface optimises for one recommendation at a time. Location scouting requires parallel consideration.

A chatbot felt modern and AI-forward on the surface, but it would have replaced structure that filmmakers still depend on:

— Structured browsing to visually compare multiple options
— Manual control over discovery — not being funnelled toward a single output.
My decision:
Design AI as an enhancement layer on top of a structured workflow, not a replacement for it. Intelligence without removing control.

04 Designed for Trust

I built explainability directly into the recommendation interface through three patterns:

  • 01

    Why this recommendation?

    Each result included a contextual breakdown: visual tone similarity, architectural relevance, environmental fit, production practicality. Users could see what the system was evaluating, not just what it returned.

  • 02

    Confidence Indicators

    A visible signal of how strong or uncertain the match was. A low-confidence result shown clearly is more trustworthy than a confident-looking result with no basis.

  • 03

    Similar Alternatives

    Rather than pushing one result, the system encouraged exploration. This reduced the pressure on any single recommendation to be perfect.

05 System Thinking

Beyond the UI, I designed for the full range of system states — not just the happy path.
— Low-confidence matches: what does the interface show when the AI is uncertain?
— Empty result handling: fallback logic rather than dead ends
— Structured AI reasoning: how to surface the logic without overwhelming the interface.
— Scalable patterns: recommendation structure built for future data and use cases

04 Outcomes

The platform launched in beta with a curated cohort of Lagos-based vendors and production companies. Early signals pointed to measurable improvements in time-to-booking and vendor conversion from browsing sessions.
